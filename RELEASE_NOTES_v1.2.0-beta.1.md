# RefChecker v1.2.0-beta.1 Release Notes

Release date: 2026-05-26

This beta focuses on making RefChecker easier to use from Claude Desktop, Claude 网页版, and other normal web pages, while removing the old LLM-generated explanation workflow.

## Highlights

### Desktop app simplification

- Removed the LLM-related generated-explanation workflow from the desktop app, backend, reports, CSV output, and docs.
- Removed the desktop top menu bar.
- Kept the main workflow focused on local/reference-database verification evidence, repair suggestions, and final human review.

### Claude Desktop MCP

- Added a local MCP server so Claude Desktop can call RefChecker directly.
- This lets users check references without switching back and forth between Claude Desktop and the RefChecker app.
- See `MCP_CLAUDE_DESKTOP.md`.

### Browser extension for Claude web and normal web pages

- Added a Chrome / Edge extension that talks to the local RefChecker HTTP bridge.
- The extension works on ordinary `http://` / `https://` pages, including Claude 网页版, subject to normal browser extension restrictions.
- The right-side result panel is injected into the page through an isolated Shadow DOM container.
- Added a switch to enable/disable the green floating “RefChecker 核查” button.
- Added source-priority chips so users can choose sources without manually typing comma-separated names.
- Updated the browser extension icon to use the same RefChecker app logo.

### Pasted full-link checking

- Added “粘贴链接核查” for checking copied full URLs/DOIs.
- This is the recommended workflow for AI pages where the visible link text looks right but the real target points to another paper.
- Users should right-click the link, copy the complete link address, paste it into the extension panel, then run the check.

### Removed short-label/hidden-link auto-detection

- Selected-text checking no longer tries to infer hidden links behind short labels such as `arXiv` or `PNAS`.
- This avoids false positives, page-specific DOM failures, and stuck “正在核查” states caused by fragile hidden-link scraping.
- The two intended workflows are now:
  1. Select a complete reference entry → check selected text.
  2. Copy a complete URL/DOI → paste it into the extension panel → check pasted link.

## Packaging / verification performed

Before packaging this release, the following checks should pass:

```powershell
node --check .\browser_extension\refchecker_claude\background.js
node --check .\browser_extension\refchecker_claude\content.js
node --check .\browser_extension\refchecker_claude\popup.js
python -m json.tool .\browser_extension\refchecker_claude\manifest.json
python -m compileall -q refchecker\link_context.py refchecker\http_server.py
python -m pytest tests\test_http_server.py tests\test_mcp_server.py
```

The Windows portable package is generated by:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tool\package_windows.ps1 -Version 1.2.0-beta.1
```

When only extension/docs changed and existing backend/frontend binaries are already built, packaging can use:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tool\package_windows.ps1 -Version 1.2.0-beta.1 -SkipBackend -SkipFlutter
```
